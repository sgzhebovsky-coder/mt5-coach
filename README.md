# MT5 Trade Coach — автоматический разбор сделок на почту

Как это работает: твой MT5-аккаунт (тот же, в который ты заходишь с телефона) подключается
к облачному сервису MetaApi — он сам "видит" историю сделок прямо с сервера брокера, поэтому
никакой локальный терминал/компьютер не нужен. Раз в 15 минут GitHub бесплатно запускает
скрипт, который проверяет новые закрытые сделки и шлёт письма на твою почту:

- после каждой закрытой сделки — короткий разбор;
- после каждой 5-й сделки — сводка по последним 5;
- после каждой 10-й сделки — более глубокий разбор (психология, дисциплина, тильт).

## Шаг 1. MetaApi (подключение к счёту)

1. Зарегистрируйся на https://app.metaapi.cloud (бесплатно, доступ к 1 счёту — без оплаты).
2. Возьми API-токен на странице https://app.metaapi.cloud/token — это `METAAPI_TOKEN`.
3. На странице https://app.metaapi.cloud/accounts нажми "Add account" и укажи логин,
   пароль и сервер от твоего MT5-счёта (точно как при входе в мобильном MT5).
   Если брокер выдаёт инвесторский (read-only) пароль — лучше используй его, тогда сервис
   физически не сможет торговать твоими деньгами, только читать историю.
4. Подожди 2–5 минут, пока статус счёта станет "deployed"/"connected".
5. Скопируй ID счёта из той же страницы — это `METAAPI_ACCOUNT_ID`.

## Шаг 2. Gmail (отправка писем)

1. Включи двухфакторную аутентификацию в своём Google-аккаунте (без неё пароль приложения не создать).
2. Зайди на https://myaccount.google.com/apppasswords и создай пароль приложения.
3. Это и есть `GMAIL_APP_PASSWORD`. Обычный пароль от Gmail для этого не подойдёт.
4. `GMAIL_ADDRESS` — твоя почта gmail, с которой будут уходить письма.
5. `RECIPIENT_EMAIL` — куда присылать отчёты (можно ту же почту).

## Шаг 3. GitHub (автоматический запуск)

1. Создай новый репозиторий на https://github.com (можно приватный).
2. Залей туда все файлы из этой папки (через веб-интерфейс "Add file → Upload files"
   или `git push`, если умеешь пользоваться git).
3. Зайди в Settings → Secrets and variables → Actions → New repository secret и добавь
   5 секретов: `METAAPI_TOKEN`, `METAAPI_ACCOUNT_ID`, `GMAIL_ADDRESS`, `GMAIL_APP_PASSWORD`,
   `RECIPIENT_EMAIL`.
4. Всё. Workflow в `.github/workflows/trade-review.yml` запустится сам по расписанию.
   Чтобы проверить сразу, не дожидаясь 15 минут — открой вкладку Actions →
   "MT5 Trade Review" → "Run workflow".

## Проверка перед тем как доверять системе

Перед тем как полностью на это положиться, рекомендую:

1. Прогнать скрипт локально один раз (`pip install -r requirements.txt`, прописать
   переменные окружения и запустить `python trade_review.py`), посмотреть, что письмо
   реально приходит и цифры в нём похожи на правду.
2. Закрыть тестовую сделку (можно совсем маленьким лотом) и убедиться, что письмо пришло.

## Важные ограничения, о которых нужно знать

- **Это не "в реальном времени"** — задержка до 15 минут (зависит от расписания cron).
  Можно уменьшить интервал, но GitHub не гарантирует точность запуска по минутам.
- **Лимит бесплатных минут GitHub Actions** для приватных репозиториев — около 2000 минут
  в месяц. При интервале 15 минут это, скорее всего, впишется, но если будет мало —
  смени `*/15` на `*/30` в файле workflow.
- **MetaApi бесплатный тариф** — доступ к одному торговому счёту без оплаты.
- **Частичные закрытия позиции** считаются как отдельные "сделки" — если ты закрываешь
  позицию по частям, статистика по count может немного отличаться от твоего внутреннего счёта.
- **Расчёт риска в % от баланса** — оценочный: он работает только если у сделки был
  выставлен стоп-лосс на момент входа и брокер отдаёт данные по тик-вэлью инструмента.
  Если где-то будет "н/д" — значит этих данных не хватило, это не баг.
- Если структура данных твоего брокера немного отличается (бывает у некоторых ДЦ),
  какие-то поля письма могут показывать "н/д" — пришли мне пример вывода в консоли
  (`print`) при локальном запуске, и я подправлю под твой случай.

## Файлы в этом проекте

- `trade_review.py` — основной скрипт анализа и отправки писем.
- `requirements.txt` — зависимости Python.
- `.github/workflows/trade-review.yml` — расписание автозапуска.
- `state.json` — создаётся автоматически, хранит "что уже разобрано" (не трогай руками).
